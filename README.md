# 🎬 FILM.TV — Decentralized P2P Watch Party

**FILM.TV** — это полностью децентрализованная платформа для совместного просмотра фильмов, сериалов и прослушивания музыки. В отличие от традиционных стримингов, здесь нет центрального сервера для хранения контента или данных пользователей. Всё работает на базе P2P-технологий прямо в браузере.

## 🚀 Основные возможности

- **Совместный просмотр (Watch Parties):** Создавайте комнаты и смотрите контент синхронно с друзьями. Пауза, перемотка и воспроизведение синхронизируются между всеми участниками мгновенно.
- **P2P Стриминг:** Контент раздается через WebTorrent. Чем больше людей смотрит, тем быстрее и стабильнее поток.
- **Децентрализованная БД:** Все данные о комнатах, пользователях и плейлистах хранятся в графовой базе данных **Gun.js**, которая синхронизируется между браузерами без участия бэкенда.
- **Мгновенная синхронизация:** Использование **Trystero (прямой P2P)** для команд управления плеером позволяет избежать задержек, свойственных традиционным базам данных.
- **Музыка и Видео:** Поддержка веб-стандартов (MP4, WebM, MP3, FLAC). Интеграция с метаданными (TMDB, Spotify).

## 🛠 Технологический стек

- **Core:** Pure JavaScript + [HTM](https://github.com/developit/htm) (JSX-like syntax without build step).
- **UI Component:** [Preact](https://preactjs.com/) для реактивного интерфейса.
- **P2P Data:** [Gun.js](https://gun.eco/) — децентрализованная база данных.
- **P2P Network:** [Trystero](https://github.com/dmotz/trystero) — прямое соединение между браузерами через WebRTC/BitTorrent.
- **P2P Streaming:** [WebTorrent](https://webtorrent.io/) — стриминг торрентов через Service Workers.
- **Styling:** CSS3 с эффектами Glassmorphism.


## 🔧 Запуск проекта

Для работы **Service Workers** и **WebRTC** браузер требует защищенное соединение. Проект будет работать только через:
- `http://localhost` (считается безопасным исключением)
- Любой домен с **HTTPS**

Если вы запустите проект и откроете его по IP (например, `http://192.168.1.10`), браузер **заблокирует** Service Worker и P2P-соединения.

## 📡 Как это работает (Архитектура)

### Синхронизация плеера
Мы используем гибридную систему:
1. **Gun.js** хранит "эталонное" состояние комнаты (плейлист, текущий трек).
2. **Trystero Actions** рассылают мгновенные команды (Play/Pause/Seek) напрямую между пирами.
3. **Drift Compensation:** Система автоматически вычисляет разницу в системных часах пользователей (`senderTime`) и корректирует позицию плеера с точностью до 0.1 сек.

### Стриминг контента
WebTorrent клиент инициализирует локальный сервер внутри Service Worker. Браузерный тег `<video>` делает обычный HTTP-запрос к SW, который на лету собирает файл из кусочков, полученных от других пиров.
